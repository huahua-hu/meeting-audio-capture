# MeetingAudioCapture

一个开源、仅在本地运行的 macOS 菜单栏应用。它通过 ScreenCaptureKit 录制系统声音和麦克风，并保存为一个包含混音与独立来源音轨的 MP4 文件，适用于会议、面试和访谈复盘。

[English](README.en.md)

## 功能

- 单个 MP4 内保存默认混音、系统声音和麦克风三个音轨
- 两路实时电平显示
- 暂停、继续和停止
- 录音仅保存在本地，成功导出后自动删除临时源文件
- 不联网、无遥测、无云端处理、无需虚拟声卡
- macOS 菜单栏界面，不显示 Dock 图标

## 系统要求

- macOS 15.0 或更高版本
- Xcode 16 或更高版本
- 当前已在 macOS 15.5（24F74）编译；macOS 26 仍需在对应设备上完成手工验证

## 构建与运行

```bash
make test
make app
make run
```

生成的应用位于 `.build/MeetingAudioCapture.app`。本项目当前采用本地临时签名，仅发布源码，不提供 Apple 公证安装包。

## 使用

1. 启动应用并在菜单栏点击波形图标。
2. 首次启动默认显示英文；在菜单底部的 `Language` 中选择 `简体中文` 可立即切换，应用会记住选择。
3. 选择麦克风与保存目录。
4. 点击 **开始录音**。
5. 根据系统提示授予“麦克风”和“录屏与系统录音”权限；首次授权后可能需要重新启动应用。
6. 确认两条电平表均有响应，再开始会议。
7. 停止后等待应用完成音频转换，然后在 Finder 中显示录音文件。

每次录制只生成一个文件：

```text
Meeting-YYYYMMDD-HHmmss.mp4
```

普通播放器默认播放 `Mixed` 混音音轨。文件内部还包含 `System Audio` 和 `Microphone` 两个独立来源音轨，可由 VLC、ffmpeg 等支持多音轨的工具识别和提取；当前版本暂不提供应用内拆分功能。录制过程中的 CAF/M4A 源文件位于 macOS 系统临时目录，MP4 成功生成后会自动删除。

系统音轨会包含所有系统播放声音，包括通知音。正式会议前建议开启勿扰模式、关闭无关音频应用，并使用耳机降低回声。

## 权限排查

在“系统设置 → 隐私与安全性”中确认 MeetingAudioCapture 已获得：

- 麦克风
- 录屏与系统录音

如果电平表一直显示 `No signal`，请检查会议软件的输出设备、所选麦克风及上述权限，然后重启应用。

## 隐私与合规

MeetingAudioCapture 不上传、分析或分享录音，也不尝试绕过 macOS、会议软件或设备管理工具的录制提示与检测。第三方会议软件的行为可能改变，本项目不保证录音不可被检测。

录制前请遵守所在地法律、会议规则、雇主政策和保密义务，并在需要时取得所有参与者同意。不要传播包含他人隐私、个人信息或商业秘密的录音。
